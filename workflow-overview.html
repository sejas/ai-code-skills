What is the Dev Workflow Plugin?

A sophisticated Claude Code plugin developed by Antonio Sejas that supercharges your development workflow with intelligent automation, Git integration, and intent-driven development practices. It bridges the gap between AI assistance and professional development practices.

Available Hooks

‚úîÔ∏ècommit-validator.sh

PreToolUse
Runs before tool execution

Validates git commits before they're executed. Enforces conventional commit prefixes (feat, fix, docs, etc.) and checks message length.

  • Enforces conventional commit format
  • Maximum 75 characters per message
  • Blocks force commits (-f flag)

üé®auto-format.sh

PostToolUse
Runs after Write/Edit operations

Automatically formats your code after making changes. Supports TypeScript/JavaScript (Prettier) and PHP (PHPCBF).

  • Prettier for JS/TS files
  • PHPCBF for PHP files
  • Runs silently in background

üé§speak-commit.sh

PostToolUse
Runs after commits

Text-to-speech announcements for commits. Get audible feedback with contextual messages based on commit type.

  • Speaks repo name and commit message
  • Contextual announcements
  • Non-blocking background execution

üìçsession-context.sh

SessionStart
Runs when session begins

Injects Git context at the start of each session. Shows uncommitted changes, current branch, and recent commits.

  • Shows uncommitted changes
  • Displays current branch
  • Lists last 5 commits

üíæsave-summary-basic.sh

SessionEnd
Runs when session ends

Simple session logging without AI summarization. Lightweight alternative for basic activity tracking.

  • Simple logging output
  • Lightweight and fast
  • Output to ~/.claude/session-log.txt

üîînotify.sh

Notification
Sends macOS notifications

Desktop notifications when Claude needs attention (permissions, waiting for input). Requires terminal-notifier.

  • Permission request notifications
  • Idle prompt alerts
  • Authentication notifications

üì±remote-notify.sh

Notification
Remote notifications

Send notifications to your phone or other devices. Supports Telegram and ntfy.sh with more backends coming.

  • Telegram integration
  • ntfy.sh support
  • Multiple backends simultaneously

ü§ñsave-summary.py

SessionEnd
AI-powered summaries

Python implementation for intelligent session summaries using the Anthropic SDK. Generates contextual activity reports.

  • AI-powered summarization
  • Requires Python 3 + claude-agent-sdk
  • Output to ~/.claude/session-logs/

üì¶Installation & Configuration

‚ö° One-Command Install: All hooks and skills are auto-configured when installed. 
claude plugin install sejas/ai-code-skills

Plugin Configuration

Your plugin configuration is in: .claude-plugin/plugin.json

Configured Hook Types:

  • PreToolUse: commit-validator.sh (validates commits)
  • PostToolUse: auto-format.sh, speak-commit.sh
  • SessionStart: session-context.sh (Git context)
  • SessionEnd: save-summary-basic.sh (session logging)
  • Notification: notify.sh, remote-notify.sh
